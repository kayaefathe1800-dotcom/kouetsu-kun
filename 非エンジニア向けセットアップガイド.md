# 校閲くん　かんたんセットアップガイド
### ～パソコンが苦手な方でも10分でできます～

---

## はじめに

このガイドは **プログラミングの知識がない方** でも、校閲くんを使えるようにするための手順書です。

**必要なもの**
- パソコン（Windows/Mac どちらでも OK）
- Google アカウント（Gmail をお持ちであれば OK）
- インターネット接続

**費用：すべて無料です**

---

## 全体の流れ（5ステップ）

```
① Googleアカウントで Gemini のAPIキーを取得（5分）
        ↓
② GitHub にアカウントを作成（3分）
        ↓
③ アプリのコードを GitHub にアップロード（2分）
        ↓
④ Vercel でアプリを公開（3分）
        ↓
⑤ 完成！ブラウザで使えるようになります
```

---

# STEP 1：Gemini API キーを取得する

> AIの「頭脳」を借りるための鍵です。無料で取得できます。

---

**1-1. Google AI Studio を開く**

ブラウザで以下のURLを開いてください。

```
https://aistudio.google.com/
```

---

**1-2. Google アカウントでログイン**

「Sign in with Google」または「Googleでログイン」をクリックし、
お持ちの Gmail アカウントでログインしてください。

---

**1-3. API キーを作成する**

ログイン後、左側のメニューに **「Get API key」** というボタンがあります。

① 「Get API key」をクリック
② 「Create API key」（または「APIキーを作成」）をクリック
③ プロジェクトを選ぶ画面が出た場合は「Create API key in new project」を選択

---

**1-4. キーをコピーして保存する**

`AIzaSy...` から始まる長い文字列が表示されます。

① 文字列の右にある 📋 コピーボタンをクリック
② メモ帳や Google Keep などに貼り付けて保存しておく

> ⚠️ このキーは他人に見せないでください。
> 後で Vercel に貼り付けるので、保存しておくことが重要です。

---

# STEP 2：GitHub アカウントを作成する

> コードを保管するための場所（クラウド上の引き出し）です。

---

**2-1. GitHub を開く**

```
https://github.com/
```

---

**2-2. アカウントを作成する**

① 右上の「Sign up」をクリック
② メールアドレス・パスワード・ユーザー名を入力
③ 「I'm not a robot」の確認をして「Create account」
④ 登録したメールアドレスに確認コードが届くので入力

---

**2-3. 新しいリポジトリ（保管場所）を作る**

① ログイン後、右上の「＋」ボタン → 「New repository」をクリック
② 以下のように入力：
   - Repository name（名前）：`proofreading-app`
   - Visibility（公開設定）：「Private」を選択（非公開にする）
③ 「Create repository」をクリック

> 次のページに `git remote add origin https://github.com/...` のような画面が出ます。
> **そのまま閉じないで**、次のステップに進んでください。

---

# STEP 3：コードを GitHub にアップロードする

> ここだけ少し難しいですが、コピペで OK です。

---

**3-1. Git をインストールする（まだの方のみ）**

お使いのパソコンに Git がインストールされているか確認します。

**Windowsの方：**
① キーボードの「Windowsキー」を押して「cmd」と入力 → コマンドプロンプトを開く
② `git --version` と入力して Enter
③ `git version 2.xx.x` のように表示されれば OK
④ 何も表示されない場合は https://git-scm.com/ からインストール

**Macの方：**
① 「ターミナル」を開く（Spotlight検索で「ターミナル」と入力）
② `git --version` と入力して Enter
③ バージョンが表示されれば OK

---

**3-2. コマンドプロンプト（またはターミナル）でフォルダに移動する**

**Windowsの方：**
```
cd C:\ClaudeTest\proofreading-app
```

**Macの方（Macにファイルをコピーした場合）：**
```
cd ~/Desktop/proofreading-app
```

---

**3-3. GitHub にアップロードする**

以下のコマンドを **1行ずつ** コピペして Enter を押してください。

```
git init
```
（Enterを押す）

```
git add .
```
（Enterを押す）

```
git commit -m "first commit"
```
（Enterを押す）

```
git branch -M main
```
（Enterを押す）

次のコマンドは **あなたのGitHubユーザー名** に置き換えてください：
```
git remote add origin https://github.com/【あなたのユーザー名】/proofreading-app.git
```
（例：ユーザー名が `yamada-taro` の場合）
```
git remote add origin https://github.com/yamada-taro/proofreading-app.git
```
（Enterを押す）

```
git push -u origin main
```
（Enterを押す）

> GitHub のユーザー名とパスワードを聞かれた場合は入力してください。
> パスワードは入力しても画面に表示されません（正常です）。

---

**3-4. アップロードを確認する**

GitHub のページ（`https://github.com/あなたのユーザー名/proofreading-app`）を開き、
ファイルが表示されていれば成功です。

---

# STEP 4：Vercel でアプリを公開する

> ここが一番重要です。アプリをインターネット上に公開します。

---

**4-1. Vercel を開く**

```
https://vercel.com/
```

---

**4-2. GitHub アカウントでログインする**

① 「Sign Up」または「Log In」をクリック
② 「Continue with GitHub」をクリック
③ GitHub との連携を許可する

---

**4-3. 新しいプロジェクトを作成する**

① ダッシュボードで「Add New...」→「Project」をクリック
② 「Import Git Repository」の画面が表示される
③ 先ほど作成した「proofreading-app」が一覧に表示されるので「Import」をクリック

---

**4-4. 環境変数を設定する（最重要）**

「Configure Project」という設定画面が開きます。

画面の下の方に **「Environment Variables」（環境変数）** という欄があります。

以下の順番で入力します：

**① Gemini API キーを入力する**

- 「Name」欄に：`GEMINI_API_KEY`
- 「Value」欄に：STEP 1 でコピーしたキー（`AIzaSy...` から始まる文字列）
- 「Add」ボタンをクリック

> メール送信機能も使う場合は ② ③ も入力してください
> （使わない場合はスキップしてかまいません）

**② Resend の API キー（メール送信用・任意）**

- 「Name」欄に：`RESEND_API_KEY`
- 「Value」欄に：Resend で取得したキー
- 「Add」ボタンをクリック

**③ 送信元メールアドレス（メール送信用・任意）**

- 「Name」欄に：`RESEND_FROM_EMAIL`
- 「Value」欄に：`noreply@あなたのドメイン.com`
- 「Add」ボタンをクリック

---

**4-5. デプロイする**

① 「Deploy」ボタンをクリック
② 画面が切り替わり、ビルドが始まります（2〜3分かかります）
③ 「Congratulations!」と表示されたら完成です！

---

**4-6. アプリのURLを確認する**

デプロイ完了後、以下のようなURLが発行されます：

```
https://proofreading-app-あなたのユーザー名.vercel.app
```

このURLをブックマークして、毎回このページにアクセスして使います。

---

# STEP 5：アプリを使ってみる

---

**5-1. アプリを開く**

STEP 4 で発行されたURLをブラウザで開きます。

---

**5-2. 記事を校閲する**

① 「テキスト入力」または「URLから取得」を選ぶ

   **テキスト入力の場合：**
   WordPress の記事編集画面から本文をコピーして、テキスト欄に貼り付ける

   **URLから取得の場合：**
   公開済みの記事URLを入力して「取得」をクリック

② 「狙っているキーワード」を入力（例：外壁塗装 大阪北区）

③ 「目標文字数」を入力（例：1500字前後）

④ 「校閲する」ボタンをクリック

⑤ 30秒〜1分後に結果が表示されます

---

**5-3. 結果の見方**

| タブ名 | 内容 |
|--------|------|
| 訂正一覧 | 修正が必要な箇所の一覧（元の文章・訂正後・理由・カテゴリ） |
| タイトル案 | 改善されたタイトルの候補3つ |
| 修正後全文 | 修正済みの記事全文（コピーボタンで一括コピー） |
| 最終確認 | AIによる総合評価コメント |

---

**5-4. 結果をダウンロード・共有する**

**CSV でダウンロード（Googleスプレッドシートに貼れます）：**
① 「訂正一覧」タブを開く
② 「CSVダウンロード」ボタンをクリック
③ ダウンロードされた .csv ファイルを Google スプレッドシートで開く

**メールで送信する：**
① 右上の「メールで送信」ボタンをクリック
② 送信先メールアドレスを入力（複数人は改行で区切る）
③ 「送信する」ボタンをクリック

---

# よくあるトラブルと対処法

---

**Q. STEP 3 でエラーが出た（`git: command not found` など）**

A. Git がインストールされていません。
https://git-scm.com/downloads からインストールしてください。

---

**Q. `git push` でユーザー名・パスワードを求められてうまくいかない**

A. GitHub は2021年以降、パスワードの代わりに「トークン」が必要です。

① GitHub にログイン → 右上のアイコン → 「Settings」
② 左メニュー一番下「Developer settings」
③ 「Personal access tokens」→「Tokens (classic)」→「Generate new token」
④ 「Note」に適当な名前を入力、「repo」にチェック → 「Generate token」
⑤ 表示されたトークンをコピー（一度しか表示されません）
⑥ パスワードの代わりにこのトークンを貼り付ける

---

**Q. Vercel の環境変数を後から変更したい**

A. Vercel ダッシュボード → プロジェクトを選択 → 「Settings」→「Environment Variables」
から変更できます。変更後は「Deployments」→「Redeploy」で再デプロイが必要です。

---

**Q. 記事が長くてエラーになる**

A. 8000文字を超える記事は前半・後半に分けて校閲してください。

---

**Q. URLから記事を取得できない**

A. テキスト貼り付けモードをお使いください。
WordPress の記事編集画面でテキストをコピーして貼り付けてください。

---

# メール送信機能を使いたい方へ（Resend の設定）

---

**Resend のアカウント作成**

① https://resend.com/ を開く
② 「Get Started for Free」でアカウント作成（Google アカウントで登録可）
③ ダッシュボード左メニュー「API Keys」→「Create API Key」
④ 名前を入力して「Add」→ 表示されたキーをコピーして保存

**Vercel に設定する**

Vercel ダッシュボード → プロジェクト → Settings → Environment Variables に追加：
- `RESEND_API_KEY`：上でコピーしたキー
- `RESEND_FROM_EMAIL`：`onboarding@resend.dev`（無料プランではこのアドレスのみ使用可）

> 独自ドメインを持っている場合は Resend の「Domains」で設定すると
> 自分のドメインから送信できます
